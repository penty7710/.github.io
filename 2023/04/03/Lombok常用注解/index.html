lombok常用注解

lombok可以帮自动生成一些方法，简化我们的开发。

导入依赖

@Getter/@Setter

自动生成Getter/Setter方法

@ToString

自动生成ToString方法

@EqualsAndHashCode

自动生成 equals(Object other) 和 hashcode() 方法，包括所有非静态变量和非 transient 的变量。

Q : 为什么只有一个整体的 @EqualsAndHashCode 注解，而不是分开的两个 @Equals 和 @HashCode？

A : 在 Java 中有规定，当两个对象 equals 时，他们的 hashcode 一定要相同，反之，当 hashcode 相同时，对象不一定 equals。所以 equals 和 hashcode 要一起实现，免得发生违反 Java 规定的情形发生。

如果某些变量不想要加进判断，可以透过 exclude 排除，也可以使用 of 指定某些字段

@NoArgsConstructor/@AllArgsConstructor/@RequiredArgsConstructor

自动生成构造器，差别在于参数不同

@NoArgsConstructor : 生成一个无参的构造器

@AllArgsConstructor : 生成一个包含所有参数的构造器

@RequiredArgsConstructor : 生成一个包含 “特定参数” 的构造器，特定参数指的是那些有加上 final 修饰词 或者 @NonNull 的变量们

@Data

整合包，只要加了 @Data 这个注解，等于同时加了以下注解

  • @Getter/@Setter
  • @ToString
  • @EqualsAndHashCode
  • @RequiredArgsConstructor

@Builder

可以链式编程，不用写一堆的setter方法

@Slf4j

用来打印日志
