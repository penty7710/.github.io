docker安装elasticsearch

发表于 2022-05-29

安装Elasticsearch

拉取镜像

我安装的是7.6.2版本，可以到hub.docker.com上查看支持的版本

docker pull elasticsearch:7.6.2

启动Elasticsearch

docker run -d --name elascticsearch -p 9200:9200 -p 9300:9300 -e ES_JAVA_OPTS="-Xms64m -Xmx512m" -e "discover.type=single-node" elasticsearch:7.6.2
  • 9300端口是使用tcp客户端连接使用的端口；9200端口是通过http协议连接es使用的端口；
  • -e ES_JAVA_OPTS=”-Xms64m -Xmx512m” 是配置内存大小，如果不配置，我的虚拟机因为内存小，会变得很卡
  • -e “discover.type=single-node” 单节点启动

开放端口

firewall-cmd --permanent --add-port=9200/tcp

安装kibana

拉取镜像

kibana和Elasticsearch的关系类似于navicat和mysql的关系。kibana可以查看Elasticsearch中的数据，并且提供一些语法提示。

安装的kibana版本需要和Elasticsearch的版本一致

docker pull kibana:7.6.2

启动kibana

docker run -d --name kibana -p 5601:5601 kibana:7.6.2

将kibana和Elasticsearch进行关联

由于kibana和elasticsearch是运行在docker中的，因此两个之间是相互独立的，没有关联，由于Elasticsearch的9200端口和宿主机的9200端口进行了映射，因此我们可以通过宿主机的9200找到Elasticsearch。

#进入kibana
docker exec -it kibana:7.6.2

#进入config文件夹
cd config

#修改kibana.yml,注意没有vim指令
vi kibana.yml
#将elasticsearch.hosts 修改为当前的主机号和端口号
elasticsearch.hosts: [ "http://192.168.217.4:9200" ]

重启kibana

docker restart kibana

开放端口

firewall-cmd --permanent --add-port=5601/tcp

一键启动kibana和Elasticsearch

编写composs

当我们重新启动一个kibana的容器，都需要重新将他与elasticsearch关联起来。

同时两个服务也是需要一同启动的因此我们编写一个compose，同时启动两个服务，并且写好配置，将kibana的kibana.yml与宿主机的配置文件进行关联，这样就不需要每次都修改kibana内部的文件。

复制kibana配置文件

将配置文件复制到宿主机

#创建文件夹
mkdir /opt/es-kibana
cd /opt/es-kibana
#复制文件
docker cp kibana:/usr/share/kibana/config/kibana.yml /opt/es-kibana
#修改文件内容，因为下面配置了compose，所以可以直接写网络名称
elasticsearch.hosts: [ "http://elasticsearch:9200" ]

编写Compose文件

#编写文件
vim docker-compose.yml
version: "3.3"													#指定语法的版本格式
volumes:
  data:
  config:
  plugin:
networks:
  es:
services:														#定义服务
   elasticsearch:												#服务名称
     image: elasticsearch:7.6.2									#镜像
     ports:														#端口映射
        - "9200:9200"
        - "9300:9300"
     networks:                                                  #网络
        - "es"
     environment:												#容器内部环境变量
        - "discovery.type=single-node"
        - "ES_JAVA_OPTS=-Xms64m -Xmx512m"
     volumes:
        - data:/usr/share/elasticsearch/data                    #挂载数据卷
        - config:/usr/share/elasticsearch/config
        - plugin:/usr/share/elasticsearch/plugins

   kibana:
    image: kibana:7.6.2
    ports:
        - "5601:5601" 
    networks:
        - "es" 
    volumes:
        - /opt/kibana_config/kibana.yml

安装docker-compose

curl -L https://get.daocloud.io/docker/compose/releases/download/1.25.0/docker-compose-`uname -s`-`uname -m` > /usr/local/bin/docker-compose

chmod +x /usr/local/bin/docker-compose

docker-compose -v

启动

#在 docker-compose.yml的当前路径下
docker-compose up -d
